## 安装和启动

### Q: 如何安装 MarkdownMonkey？

**A**:

1. 访问 GitHub Releases 页面
2. 下载对应系统的安装包
3. 运行安装程序
4. 完成安装后直接启动

### Q: 支持哪些操作系统？

**A**:

- ✅ Windows 10/11
- ✅ macOS 10.15+
- ✅ Linux (Ubuntu, Fedora, etc.)

### Q: 安装后无法启动怎么办？

**A**:

1. 检查系统权限
2. 尝试以管理员身份运行
3. 查看错误日志
4. 重新安装

## 文件管理

### Q: 支持哪些文件格式？

**A**:

- ✅ `.md` - Markdown
- ✅ `.markdown` - Markdown
- ⚠️ 其他格式可能无法正确预览

### Q: 如何打开文件？

**A**: 三种方式

1. 点击菜单栏 `打开` 按钮
2. 拖拽文件到编辑器
3. 从文件树双击文件

### Q: 文件自动保存吗？

**A**:
❌ 目前不支持自动保存，请手动保存（`Ctrl+S`）

### Q: 如何找到最近打开的文件？

**A**:
在设置 → UI 设置中，有 `最近文件` 列表

### Q: 可以打开整个文件夹吗？

**A**:
✅ 可以。点击 `打开文件夹`，选择文件夹作为工作区。文件树会显示该文件夹的所有文件。

## 编辑功能

### Q: 支持哪些 Markdown 语法？

**A**:
支持完整的 Markdown 语法，包括：

- 标题、列表、引用
- 粗体、斜体、删除线
- 代码块（语法高亮）
- 表格
- 链接、图片
- Mermaid 图表

### Q: 如何插入图片？

**A**:

```markdown
![图片描述](path/to/image.png)
```

- 可以使用相对路径
- 可以使用绝对路径
- 可以使用网络 URL

### Q: 代码块如何高亮？

**A**:

````markdown
```python
def hello():
    print("Hello")
```
````

指定语言后自动语法高亮。

### Q: Mermaid 图表不显示？

**A**:

1. 确认语法正确
2. 检查代码块标记为 ````mermaid`
3. 刷新预览（关闭重新打开文件）

### Q: 如何搜索和替换？

**A**:

1. 点击菜单栏 `搜索/替换`
2. 输入查找内容
3. 可选：启用正则、忽略大小写
4. 点击 `全部替换`

## 标签管理

### Q: 如何打开多个标签？

**A**:
打开多个文件即可，每个文件对应一个标签。

### Q: 快捷键切换标签？

**A**:

- `Ctrl+Tab` - 下一个标签
- `Ctrl+Shift+Tab` - 上一个标签

### Q: 如何关闭标签？

**A**:
三种方式：

1. 点击标签上的 `×` 按钮
2. 按 `Ctrl+W`
3. 右键点击标签 → 关闭

### Q: 标签名太长看不全？

**A**:

- 标签会自动截断长文件名
- 鼠标悬停查看完整名称
- 显示前 200px，其余用 `...` 表示

### Q: 如何批量关闭标签？

**A**:
右键点击任意标签：

- `关闭其他` - 关闭除当前标签外的所有标签
- `关闭右侧` - 关闭当前标签右侧的所有标签
- `关闭所有标签` - 关闭所有标签

### Q: 关闭所有标签后会怎样？

**A**:
应用显示空白编辑器，可以继续创建新文件或打开文件。

### Q: 标签可以拖拽排序吗？

**A**:
❌ 目前不支持。由于 Tauri webview 的技术限制，拖拽功能暂时无法实现。

## AI 功能

### Q: 需要付费吗？

**A**:
MarkdownMonkey 应用**免费**，但 AI 服务可能收费：

- OpenAI、DeepSeek 等需要购买 API
- Ollama 完全免费（本地运行）

### Q: 哪个 AI 提供商最好？

**A**:
取决于需求：

| 提供商   | 优势     | 适合          |
| -------- | -------- | ------------- |
| OpenAI   | 质量最高 | 专业用户      |
| DeepSeek | 便宜快速 | 国内用户      |
| Kimi     | 长上下文 | 长文档处理    |
| Ollama   | 免费私密 | 个人/隐私需求 |

### Q: 如何获取 API Key？

**A**:
每个提供商不同：

- **OpenAI**: [platform.openai.com](https://platform.openai.com/)
- **DeepSeek**: [deepseek.com](https://www.deepseek.com/)
- **Kimi**: [platform.moonshot.cn](https://platform.moonshot.cn/)

注册账号后在控制台获取。

### Q: API Key 安全吗？

**A**:

- ✅ 存储在本地配置文件
- ✅ 不会上传到服务器
- ⚠️ 不要分享给他人
- ⚠️ 定期更换

### Q: 测试连接失败怎么办？

**A**:
检查：

1. API Key 是否正确（无多余空格）
2. API Base URL 格式是否正确
3. 网络连接是否正常
4. 服务商服务是否可用

### Q: AI 回答质量不好？

**A**:
尝试：

1. 更换更好的模型
2. 优化系统提示词
3. 调整温度参数
4. 提供更清晰的指令

### Q: 如何使用本地 AI？

**A**:
使用 Ollama：

1. 安装 Ollama: [ollama.com](https://ollama.com/)
2. 下载模型: `ollama pull llama3`
3. 启动服务: `ollama serve`
4. MarkdownMonkey 选择 `Ollama (本地)`

### Q: Ollama 连接失败？

**A**:
检查：

1. Ollama 是否已安装：`ollama --version`
2. 服务是否运行：`ollama serve`
3. 端口是否正确：默认 11434
4. 模型是否已下载：`ollama list`

### Q: AI 功能耗电/资源多吗？

**A**:

- 云端 AI（OpenAI 等）：**不耗本地资源**
- 本地 AI（Ollama）：**需要较多 CPU/内存**

### Q: 可以自定义 AI 操作吗？

**A**:
✅ 可以。在设置 → `提示模板` 中创建自定义提示词。

## 性能和优化

### Q: 打开大文件很慢？

**A**:
这是正常现象。优化建议：

- 分割大文件
- 使用大纲视图快速导航
- 等待未来版本的虚拟滚动优化

### Q: Mermaid 图表渲染慢？

**A**:

- 简化图表结构
- 减少节点数量
- 避免过于复杂的关系

### Q: 内存占用高怎么办？

**A**:

1. 关闭不需要的标签
2. 不要同时打开太多文件
3. 重启应用释放内存

### Q: 应用卡顿？

**A**:
可能原因：

- 文件过大
- 图表太复杂
- 系统资源不足

解决：

- 优化文档
- 关闭其他应用
- 重启应用

## 快捷键

### Q: 有哪些快捷键？

**A**:

**标签管理**:

- `Ctrl+Tab` - 下一个标签
- `Ctrl+Shift+Tab` - 上一个标签
- `Ctrl+W` - 关闭当前标签

**其他**（取决于配置）:

- `Ctrl+S` - 保存
- `Ctrl+Z` - 撤销
- `Ctrl+Y` - 重做

### Q: 可以自定义快捷键吗？

**A**:
❌ 目前不支持自定义快捷键。

### Q: Mac 上的快捷键？

**A**:
将 `Ctrl` 替换为 `Cmd`：

- `Cmd+Tab` - 切换标签
- `Cmd+W` - 关闭标签

## 故障排除

### Q: 启动后界面空白？

**A**:

1. 检查网络连接（首次启动）
2. 清除缓存
3. 重新安装

### Q: 保存失败？

**A**:
检查：

- 文件路径是否存在
- 是否有写入权限
- 磁盘空间是否足够

### Q: 预览不更新？

**A**:

1. 检查滚动同步是否开启
2. 尝试关闭重新打开文件
3. 重启应用

### Q: 中文显示乱码？

**A**:

1. 检查文件编码（应为 UTF-8）
2. 在设置中切换语言
3. 重新打开文件

### Q: 报错"Cannot find module"？

**A**:
应用文件可能损坏，尝试：

1. 重新安装
2. 检查安装路径是否完整

### Q: 数据丢失怎么办？

**A**:
建议：

- 定期备份文件
- 使用 Git 版本控制
- 启用云同步（如 OneDrive、iCloud）

### Q: 如何查看日志？

**A**:
日志文件位置（根据系统不同）：

**Windows**:

```
%APPDATA%\com.markdownmonkey.app\logs
```

**macOS**:

```
~/Library/Logs/com.markdownmonkey.app/
```

**Linux**:

```
~/.config/com.markdownmonkey.app/logs/
```

### Q: 如何重置设置？

**A**:
删除配置文件：

**Windows**:

```
%APPDATA%\com.markdownmonkey.app\
```

**macOS**:

```
~/Library/Application Support/com.markdownmonkey.app/
```

**Linux**:

```
~/.config/com.markdownmonkey.app/
```

再重启应用即可恢复默认设置。
